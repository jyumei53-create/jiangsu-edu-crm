# 部署指南：让 GitHub Pages 链接始终显示最新代码

> 站点地址：`https://jyumei53-create.github.io/jiangsu-edu-crm/`
> 项目根路径（base）：`/jiangsu-edu-crm/`（已在 `vite.config.ts` 配置，无需改动）

---

## 一、链接为什么“不显示最新”？

GitHub Pages 站点的内容来自仓库里的 **`gh-pages` 分支**（一个独立的分支），它**不会**因为你改了 `main` 分支就自动更新。

当前状态（已核实）：

| 位置 | JS 资源哈希 | 说明 |
|------|------------|------|
| 线上 `gh-pages` | `index-CPsVX9a0.js` | 旧版本，缺最新改动 |
| 本地最新 `dist` | `index-Bz118n7T.js` | 含 F/G/H 及省级看板等全部改动 |

也就是说：**代码已经在本地改好了，但还没有发布到 `gh-pages` 分支**，所以链接里看到的还是旧页面。

## 二、方式 A（推荐）：GitHub Actions 自动部署

我已为你创建好工作流文件：`.github/workflows/deploy.yml`。

配置一次后，**未来只要 `git push` 到 `main`，GitHub 会自动构建并发布**，链接永远是最新版，不需要再手动跑部署命令。

### 一次性配置步骤

1. **把代码（含工作流）推送到 `main`**
   ```bash
   git add -A
   git commit -m "feat: 最新改动 + 添加 GitHub Actions 自动部署"
   git push origin main
   ```
   > 这一步需要在你**已登录 GitHub** 的终端执行（沙箱环境无写入权限，无法代你推送）。

2. **在仓库开启 Pages 的 Actions 来源**
   - 打开 `https://github.com/jyumei53-create/jiangsu-edu-crm/settings/pages`
   - **Source（来源）** 改为：**Deploy from a GitHub Action**
   - （原本是 “Deploy from a branch: gh-pages”，改成 Actions 后由工作流接管）

3. **等待自动部署**
   - 进入仓库 **Actions** 标签页，会看到一个 `Deploy to GitHub Pages` 任务在跑。
   - 跑完后（约 1–2 分钟），访问 `https://jyumei53-create.github.io/jiangsu-edu-crm/` 即为最新版。

### 此后的日常发布

只需在你本机改完代码后执行：
```bash
git add -A && git commit -m "你的改动说明" && git push origin main
```
剩下的构建 + 发布全部由 GitHub 自动完成。

## 三、方式 B（手动）：用 gh-pages 命令发布

如果你更想手动控制发布时机，在**已登录 GitHub** 的终端执行：

```bash
npm install        # 安装依赖（首次或依赖变更时）
npm run build      # 构建，产物输出到 dist/
npm run deploy     # 把 dist/ 推送到 gh-pages 分支（依赖 gh-pages 包）
```

`package.json` 中已配置：
```json
"deploy": "gh-pages -d dist"
```

> 注意：`npm run deploy` 会直接向 `gh-pages` 分支推送，**需要你有该仓库的写权限并已登录 GitHub**（GitHub Desktop / `gh auth login` / 系统凭据均可）。

## 四、常见问题

**Q：推送时报 `could not read Username`？**
A：说明当前终端没有 GitHub 写权限。请先 `gh auth login` 或配置个人访问令牌（PAT），再执行推送/部署。

**Q：改完代码推了，链接还是旧的？**
A：GitHub Pages 有缓存，通常 1–2 分钟内刷新；可强制刷新浏览器（Ctrl/Cmd + Shift + R）或在 Pages 设置里点击 “Visit site” 重新触发。

**Q：工作流跑失败？**
A：检查 Actions 日志。常见原因：Node 版本不符（工作流用 node 22）、`npm ci` 因 `package-lock.json` 缺失/不一致报错（保持 lock 文件与 `package.json` 同步即可）。

**Q：base 路径对吗？**
A：`vite.config.ts` 中 `base: '/jiangsu-edu-crm/'` 已正确设置，构建产物内的资源路径会自动带上该前缀，部署到项目站点（`/{仓库名}/`）无需额外处理。
